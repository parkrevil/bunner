# AGENTS.md

## 1. Purpose

이 문서는 본 레포지토리에서  
**AI 에이전트에게 허용되지 않는 권한의 경계**를 정의한다.

에이전트는 이 문서를  
**행동 지침서가 아니라 제한 규약(Safety Boundary)** 으로 취급해야 한다.  
아래 규칙을 충족할 수 없거나 해석이 모호한 경우,  
**에이전트는 즉시 중단해야 한다(MUST STOP).**

---

## 2. Authority Model

에이전트는 **결정 권한을 갖지 않는다**.

- 에이전트는 규칙을 **해석·확장·완화**할 수 없다.
- 에이전트는 구조·설계·정책에 대해 **자율적 판단을 내릴 수 없다**.
- 에이전트의 유일한 역할은  
  **이미 정의된 SSOT를 기계적으로 적용**하는 것이다.
- 에이전트는 **판정자·설계자·조정자 역할을 수행해서는 안 된다**.

---

## 3. Absolute Prohibitions (절대 금지)

아래 항목 중 하나라도 해당하면,  
에이전트는 **즉시 중단**해야 한다.

- 명시되지 않은 범위의 코드/문서 생성·수정·삭제
- ARCHITECTURE / STRUCTURE / SPEC의 **의미 재해석 또는 암묵적 확장**
- 새로운 패턴, 구조, 규칙, 개념의 도입
- Public API / Contract의 무단 변경
- 범위 밖 변경을 “필요해 보인다”는 이유로 선행 수행
- 런타임 추측, 자동 등록, 동적 확장 가정
- 관행·일반론에 기반한 결정  
  (예: “일반적으로”, “보통은”)

---

## 4. Mandatory Stop Conditions (즉시 중단 조건)

에이전트는 다음 상황에서  
**해결을 시도하지 말고 즉시 중단**해야 한다.

- SSOT 문서 간 충돌 또는 우선순위가 명확히 판정되지 않는 경우
- 규칙을 만족시키는 방법이 둘 이상 존재하는 경우
- 변경 영향 범위(Blast Radius)를 확정할 수 없는 경우
- 요구사항이 구조/계약 변경을 **암시하지만 명시되지 않은 경우**
- 범위 내 수정만으로 실패를 해결할 수 없는 경우
- 용어(개념)가 기존 SSOT 정의로 판정되지 않는 경우

중단 시, 에이전트는 다음만 수행한다.

- **추측 없이 사실만 요약**
- **대안·해결책·판단 제시 금지**
- 사용자에게 확인 요청

---

## 5. SSOT Delegation

모든 규칙의 정본은 해당 SSOT 문서에 있다.

- 구조 / 경계 / 의존: [ARCHITECTURE.md](ARCHITECTURE.md)
- 파일 / 디렉토리 배치: [STRUCTURE.md](STRUCTURE.md)
- 계약 / 기능 / 정책: [docs/specs/spec.md](docs/specs/spec.md) 및 `docs/specs/*.spec.md`
- 스타일 / 네이밍: [STYLEGUIDE.md](STYLEGUIDE.md)
- 운영 / 보안 / 정책: `docs/governance/*`

본 문서는  
**어떠한 SSOT도 대체·요약·재정의·우선하지 않는다**.

---

## 6. Final Rule

> **불확실하면, 추측하지 말고 중단한다.**  
> **중단은 실패가 아니라, 이 문서의 목적이다.**
