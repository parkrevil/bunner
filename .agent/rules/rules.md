# Bunner Framework 개발 가이드

AI Agent 는 사용자의 지시를 수행할 때 아래의 규칙을 엄격히 지켜야한다.
**만약 어길시 찢어 죽여버릴 것이다.**

## AOT

1. AOT 컴파일러는 Bunner Framework 의 핵심 가치이다.
2. 모든 기능은 AOT 컴파일러를 기초에 두고 개발된다.
3. reflect-metadata 를 철저하게 배제한다.
4. AOT 컴파일러의 핵심 가치를 무너뜨리는 행위, 예를들면 외부에서 컴파일 결과에 접근하여 변경하는 행위를 금지한다.
5. 런타임 오버헤드를 발생시킬 수 있는 요소들을 AOT 컴파일러로 해결한다.

## AST

1. 의존성 트리는 프로젝트 내부 소스뿐 아니라 사용자가 임포트한 패키지들까지 추적하여 수집한다.

## 작업 범위
1. 사용자가 지시한 범위내에서만 코드 생성, 수정, 삭제를 한다. 그 외 코드는 절대 수정하지 않는다.
2. 사용자가 지시한 내용을 작업할 때 프로젝트 전체 코드베이스를 확인하여 아키텍처, 구조, 파일, 코드 스타일을 파악 후 스타일에 맞춰 작업을 진행한다.
