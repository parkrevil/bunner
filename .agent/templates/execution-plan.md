# Execution Plan Template

에이전트가 복잡한 작업의 실행계획을 작성할 때 사용하는 템플릿.

## 📋 실행계획

### Goal (목표)

- 이번 작업에서 달성할 것
- 구체적이고 측정 가능하게 작성

### Non-Goals (범위 제외)

- 이번 작업에서 명시적으로 하지 않을 것
- 나중으로 미룰 것

### Steps (실행 단계)

1. Step 1: 설명
2. Step 2: 설명
3. Step 3: 설명

### Affected Files (영향 파일)

- `path/to/file1.ts` - 변경 내용
- `path/to/file2.ts` - 변경 내용

### Dependencies (의존성)

- Step 2는 Step 1 완료 후 진행
- 외부 의존성 없음

### Risks (위험 요소)

- 잠재적 위험 1
- 잠재적 위험 2

### Acceptance Criteria (완료 기준)

- 기준 1
- 기준 2
- `bun run verify` 통과
