# Provider

Provider는 Bunner의 DI 시스템에서 의존성 그래프를 확장하고, 외부 리소스나 커스텀 로직을 주입 가능(Injectable)하게 만드는 메커니즘이다. 일반적인 서비스 클래스는 AOT 컴파일러가 파일 시스템을 통해 자동으로 발견하지만, 복잡한 생성 로직이나 외부 패키지, 상수 값 등은 Provider를 통해 명시적으로 정의된다.

## Key Philosophy & Features

- **Complement to Auto-Discovery (자동 발견의 보완):**
  - Bunner는 기본적으로 "설정 없는 연결(Zero-Config Wiring)"을 지향하므로, 표준적인 서비스는 등록 없이 사용된다.
  - Provider는 이러한 자동 발견이 불가능하거나(예: 인터페이스에 구현체 매핑), 동적인 설정이 필요한 영역(예: 팩토리, 값 주입)을 담당한다.

- **Static Definition, Dynamic Execution (정적 정의, 동적 실행):**
  - Provider의 등록 정보(토큰, 타입, 팩토리 함수 등)는 빌드 타임에 분석되어 의존성 그래프에 고정된다.
  - 실제 인스턴스 생성이나 팩토리 실행은 런타임 초기화 시점에 이루어지지만, 그 연결 구조는 이미 컴파일러에 의해 확정된 상태다.

- **Explicit Integration Point (명시적 통합 지점):**
  - 외부 패키지나 라이브러리를 애플리케이션의 DI 시스템으로 가져오는 공식적인 통로다.
  - 이를 통해 외부 라이브러리도 프레임워크의 생명주기 관리와 의존성 주입의 이점을 누릴 수 있다.
